# gnomon

On load, the clock makes a `fetch` request to `worldtimeapi.org` to get an accurate `UTC` timestamp. To compensate for network latency, it measures the round-trip time of the request and adds half of it to the server's returned time, giving a reasonable estimate of the true current moment. The difference between that corrected server time and the local `Date.now()` is stored as an offset, which gets applied on every tick so the displayed time stays accurate regardless of how far off the device's system clock might be. The hands update once per second, and the timezone conversion is handled entirely client-side using the browser's built-in `Intl` API — no additional requests are made when switching zones. The offset is re-synced every 60 seconds to prevent drift.

The date display shows three small windows side by side — one for the two-digit year, one for the month, and one for the day. Most of the time each window simply shows a still number. As a changeover approaches, the current number begins to slowly scroll upward and out of the window while the next number rises up from below to replace it, with the motion easing in and then easing out smoothly. Once the new number has settled into place, everything goes still again until the next changeover draws near. All three wheels follow this same behavior, each on their own respective cycle.
